# Characteristics

The audience understands software systems and architecture diagrams. It has
limited patience for generic marketing language and wants to know what is
true, what is inferred and what action is recommended.

# Content guidance

- Define specialized terms once, then use them consistently.
- Prefer one argument per slide.
- Show relationships and evidence before implementation detail.
- Keep claims concise and attach citations where available.
